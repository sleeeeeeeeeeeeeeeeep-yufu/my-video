# 動画生成パイプライン仕様

> **概要**: 入力動画から字幕付きショート動画を自動生成するエンドツーエンドの処理フロー

---

## 1. パイプライン全体図

```
┌─────────────────────────────────────────────────────────────────┐
│                         ユーザー操作 (UI)                         │
└─────────────────────────────────────────────────────────────────┘
         │                    │
         ▼                    ▼
  ① 台本ファイル (.txt)   ② 元動画 (MP4)
         │                    │
         │                    ▼
         │           ③ S3 / Blob ストレージへアップロード
         │                    │
         │                    ▼
         │         ④ クライアント側 音声チャンク抽出
         │            (AudioContext API, 30秒ごとに分割)
         │                    │
         │                    ▼
         │        ⑤ /api/analyze [AI処理]
         │            Whisper 音声認識
         │            → transcript 生成
         │            → episode.json 生成
         │                    │
         └───────► ⑥ 台本反映 /api/chat [AI処理]
                      (台本が存在する場合に自動実行)
                              │
                              ▼
                    ⑦ episode.json (UI State)
                              │
                              ▼
                    ⑧ Remotion レンダリング
                       /api/render (Lambda)
                              │
                              ▼
                    ⑨ MP4 出力 → ダウンロード
```

---

## 2. 各ステップの詳細

### ① 台本アップロード（`.txt` / `.md`）

| 項目   | 内容                      |
|------|--------------------------|
| 処理者 | クライアント (FileReader API) |
| 入力  | テキストファイル               |
| 出力  | `scriptText` (メモリ上の文字列) |
| 役割  | 後続の AI 字幕生成の参照原稿として保持   |

---

### ② 元動画アップロード（MP4）

| 項目   | 内容                                            |
|------|------------------------------------------------|
| 処理者 | クライアント + Next.js API Route (`/api/get-signed-url`) |
| 入力  | ユーザーが選択した MP4 ファイル                          |
| 出力  | S3 上の動画 URL (`videoSrc`)                      |
| 役割  | 元動画の永続化とRemotionへの参照URL取得                   |

- `getVideoDuration()` でブラウザ上から尺（秒数）を取得し `meta.durationInFrames` を計算する
- アップロードは XMLHttpRequest で直接 S3 にアップロード（プログレス表示対応）

---

### ③ 音声チャンク抽出

| 項目   | 内容                                      |
|------|------------------------------------------|
| 処理者 | クライアント (`src/lib/audioUtils.ts`)        |
| 入力  | 元動画ファイル (File オブジェクト)                  |
| 出力  | WAV チャンク配列 (30秒ごと)                     |
| AI処理| なし（ブラウザの AudioContext API を使用）         |

- サーバーへの動画ファイル転送を避け、音声のみを抽出してデータ量を削減する
- チャンク分割により Whisper の長さ制限を回避する

---

### ④ Whisper 音声解析 → transcript 生成

| 項目      | 内容                                           |
|---------|-----------------------------------------------|
| 処理者    | サーバー (`/api/analyze`)                        |
| 入力     | WAV チャンク配列                                  |
| 出力     | transcript (発話テキスト + タイムスタンプ)           |
| **AI処理** | ✅ Whisper による音声→テキスト変換                    |
| 時間同期  | Whisper が返す `start` / `end` (秒) → frame 変換  |

- transcript の各セグメントには音声の開始・終了時刻が付与される
- この段階が**時間同期処理の起点**となる

---

### ⑤ episode.json 生成

| 項目      | 内容                                          |
|---------|----------------------------------------------|
| 処理者    | サーバー (`/api/analyze`)                       |
| 入力     | transcript                                   |
| 出力     | `episode.json` の `segments` 配列              |
| **AI処理** | ✅ LLM による transcript → segments 変換         |
| **編集ロジック中心** | ✅ ここで字幕分割・アニメーション種別・SE の割り当てを決定 |

- LLM が transcript の内容・感情・重要度を判断し `type` / `animation` / `se` を割り当てる
- 時刻は Whisper の出力を 30fps に変換して `start` / `end` フレームとして設定する
- この段階の出力が**パイプライン全体の編集仕様**となる

---

### ⑥ 台本反映（Chat API）

| 項目      | 内容                                          |
|---------|----------------------------------------------|
| 処理者    | サーバー (`/api/chat`)                          |
| 入力     | `segments` + `scriptText` (台本)              |
| 出力     | 台本に基づいて書き換えられた `segments`             |
| **AI処理** | ✅ LLM による台本→セグメント上書き                   |

- 台本が事前にアップロードされている場合、音声解析完了後に自動的に実行される
- Whisper が生成した字幕テキストを台本の内容で置き換える（タイミングは維持）
- ユーザーが後から手動で追加指示することも可能（チャットインターフェース）

---

### ⑦ Remotion レンダリング

| 項目   | 内容                                            |
|------|------------------------------------------------|
| 処理者 | Remotion Lambda (`/api/render`)                 |
| 入力  | `episode.json` (UI State) + 元動画 URL           |
| 出力  | MP4 ファイル                                     |
| 役割  | episode.json の仕様に従って動画をフレーム単位で合成する    |

- レンダリングは Remotion Lambda で非同期実行される
- `episode.json` の全フィールドを `inputProps` として Composition に渡す
- `afterToOriginal()` により字幕タイミングが元動画フレームに逆算される

---

## 3. 各ステップの役割整理

| ステップ | 役割 | AI処理 | 時間同期 | 編集ロジック |
|---------|-----|:------:|:------:|:---------:|
| 音声チャンク抽出 | データ前処理 | - | - | - |
| Whisper 解析 | 音声→テキスト変換 | ✅ | **起点** | - |
| episode.json 生成 | 編集仕様生成 | ✅ | ✅ | **中心** |
| 台本反映 (Chat) | テキスト上書き | ✅ | - | ✅ |
| Remotion レンダリング | 動画合成 | - | **実行** | - |

---

## 4. 編集ロジックの中心

```
/api/analyze の episode.json 生成ステップ
```

- 字幕のタイミング・アニメーション種別・SE・ズーム設定がすべてここで決定される
- LLM が動画の「構造（hook → fact → conclusion）」を理解してセグメントを分類する
- ユーザーはチャットインターフェースでこの仕様を後から修正できる

---

## 5. 時間同期処理の位置

### Whisper → episode.json（秒 → フレーム変換）

```
Whisper: { start: 3.2, end: 7.8 }  // 秒単位
              ↓ × fps(30)
episode.json: { start: 96, end: 234 }  // フレーム単位
```

### episode.json → Remotion（カット後 → 元動画フレーム変換）

```
segment.start (カット後フレーム)
    ↓ afterToOriginal(frame, cuts)
Sequence.from (元動画フレーム)
```

| 変換箇所                 | 処理内容                            |
|------------------------|-----------------------------------|
| `/api/analyze`         | Whisper 秒 → episode.json フレーム  |
| `Main.tsx afterToOriginal()` | カット後フレーム → 元動画フレーム     |

---

## 6. データフロー全体図（型レベル）

```
File (MP4)
  └─ extractAudioFromVideo()
       └─ Blob[] (WAV chunks)
           └─ POST /api/analyze
               └─ Whisper API → transcript[]
               └─ LLM API    → episode.json
                    └─ UI State (inputEpisode)
                         └─ [オプション] POST /api/chat
                              └─ 修正済み segments[]
                         └─ POST /api/render
                              └─ Remotion Lambda
                                   └─ MP4 ダウンロード
```
